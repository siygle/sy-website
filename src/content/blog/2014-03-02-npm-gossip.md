看起來囉哩囉唆一大堆，不過其實就像是小孩吵架一樣的感覺 XDDD

總之，就是一開始有開發者註冊了一個新套件，然後把他取名叫做 `npmjs`，結果就被 Isaac 通知說這個名稱不行，然後把套件移除，然後 nodejitsu 就出來把 npm 註冊走了，然後發了篇文寫了些冠冕堂皇的理由，解釋為什麼要作這件事。什麼，你問為什麼 nodejitsu 突然亂入？因為那位開發者就是 nodejitsu 的員工...

巧合!? 套一句 Shelock 的名言 - **Nothing is a coincidence**(大誤)

然後就炸開了...0rz

還有後續，不過目前看起來應該是稍微平靜了，有興趣可以把上面推裡的文章看完...xDDD

nodejs 算是我第一個陷入這麼深的東西，所以一路看來其實這類的事件已經發生不少次了，不知道是因為 node 社群人人都是性情中人，還是社群本來就常會發生這樣的事，不過瞄到連 ruby 的人都拿這個來調侃，應該是過於頻繁了些。

其實[之前](http://blog.sylee.dev/next-move-of-node-js/)就有提過 npm Inc 與 nodejitsu 有點衝突的業務，看來不算是杞人憂天啊 ;p 好像到目前為止，nodejs 的生態圈都還不算是基於一個公開的組織，而是某些營利公司之下（Joyent、nodejitsu，還有最近的 npm Inc），這樣的確是會讓人有些擔心啊。

像這樣的東西偶而出現讓大家笑笑 (!?) 還好，事後如果對整體生態圈有助益的話還好，希望不要因此把整個社群搞爛掉才好...開源專案的運作，果然是一門深奧的學問啊~
